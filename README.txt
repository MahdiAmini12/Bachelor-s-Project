راهنمای پروژه مقایسه عملکرد Docker و Podman
درس: پروژه عملی
دانشجو: محمد مهدی رسول امینی
استاد راهنما: دکتر سلطانی

معرفی پروژه
این پروژه به مقایسه عملکرد دو پلتفرم کانتینرسازی Docker و Podman می‌پردازد. هدف اصلی، سنجش مصرف منابع سیستم (CPU، حافظه، شبکه) در محیط‌های مختلف و ارائه تحلیل‌های کمی است.

ساختار پوشه‌ها
text
/project_root/
│── /docs/                # مستندات پروژه
│   ├── proposal.pdf      # پروپوزال اولیه
│   └── final_report.pdf  # گزارش نهایی
│── /src/                 # کدها و اسکریپت‌ها
│   ├── run_tests.sh      # اسکریپت اجرای تست‌ها
│   └── monitoring_tools/ # ابزارهای مانیتورینگ
│── /data/                # داده‌های خام
│   ├── docker_results/   # نتایج Docker
│   └── podman_results/   # نتایج Podman
└── README.txt            # همین فایل
راهنمای اجرا
پیش‌نیازها
سیستم عامل: Ubuntu 22.04 LTS/ kali linux 2024.4

حداقل سخت‌افزار:

4GB RAM

2 هسته CPU

نصب ابزارهای مورد نیاز:

sudo apt update
sudo apt install docker.io podman sysbench stress-ng htop
مراحل اجرا
اجرای تست‌ها:

اسکریپت run_tests.sh را اجرا کنید:

chmod +x src/run_tests.sh
./src/run_tests.sh
نتایج در پوشه /data ذخیره می‌شود.

مانیتورینگ منابع:

برای مشاهده مصرف CPU:

./src/monitoring_tools/monitor_cpu.sh
برای مشاهده مصرف حافظه:

./src/monitoring_tools/monitor_mem.sh
تحلیل نتایج:

از فایل‌های خام در /data برای تحلیل استفاده کنید.

گزارش نهایی در /docs/final_report.pdf ارائه شده است.

توضیحات فایل‌ها
proposal.pdf: شامل اهداف، روش‌شناسی و ابزارهای مورد استفاده.

final_report.pdf: تحلیل نتایج، نمودارها و جمع‌بندی نهایی.

run_tests.sh: اسکریپت خودکار برای اجرای تمام تست‌ها.

داده‌های خام: شامل خروجی‌های ابزارهای تست برای هر پلتفرم.

نکات مهم
قبل از اجرا، مطمئن شوید Docker و Podman به درستی نصب شده‌اند.

برای تست‌های شبکه، ممکن است نیاز به تنظیمات فایروال داشته باشید.

فایل‌های خام در /data می‌توانند برای تحلیل‌های بیشتر استفاده شوند.
